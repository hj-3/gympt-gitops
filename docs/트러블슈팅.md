# 트러블슈팅 가이드

## Argo CD 문제

### 1. Application Out of Sync

**증상**:
```
argocd app get backend-api-dev
...
Sync Status: OutOfSync
```

**원인**:
- Git과 클러스터 상태 불일치
- HPA가 replicas 변경
- 수동으로 kubectl edit 실행

**해결**:
```bash
# 차이점 확인
argocd app diff backend-api-dev

# 동기화 (Git 상태로 복구)
argocd app sync backend-api-dev

# Self-heal 활성화
kubectl patch application backend-api-dev -n argocd \
  --type merge \
  -p '{"spec":{"syncPolicy":{"automated":{"selfHeal":true}}}}'
```

### 2. Helm Chart 배포 실패

**증상**:
```
Sync Status: Failed
Message: ComparisonError: Manifest generation error
```

**원인**:
- Helm template 문법 오류
- values.yaml 유효하지 않은 값

**해결**:
```bash
# 로컬에서 Helm template 검증
helm template backend-api charts/backend-api \
  -f charts/backend-api/values-dev.yaml \
  --debug

# Helm lint 실행
helm lint charts/backend-api

# Argo CD 로그 확인
kubectl logs -n argocd -l app.kubernetes.io/name=argocd-application-controller
```

### 3. Sync Timeout

**증상**:
```
Sync Status: Unknown
Message: context deadline exceeded
```

**원인**:
- 리소스 생성이 오래 걸림
- Health Check 실패

**해결**:
```bash
# Timeout 증가
kubectl patch application backend-api-dev -n argocd \
  --type merge \
  -p '{"spec":{"syncPolicy":{"retry":{"limit":5,"backoff":{"duration":"5s","maxDuration":"5m"}}}}}'

# Pod 상태 확인
kubectl get pods -n backend-api

# 이벤트 확인
kubectl get events -n backend-api --sort-by='.lastTimestamp'
```

## Pod 문제

### 1. ImagePullBackOff

**증상**:
```
kubectl get pods -n backend-api
NAME                           READY   STATUS             RESTARTS   AGE
backend-api-xxx                0/1     ImagePullBackOff   0          2m
```

**원인**:
- 이미지 태그 오류
- ECR 권한 부족
- ImagePullSecret 누락

**해결**:
```bash
# Pod 상세 정보
kubectl describe pod backend-api-xxx -n backend-api

# ECR 로그인 확인
aws ecr get-login-password --region ap-northeast-2

# ImagePullSecret 생성 (필요 시)
kubectl create secret docker-registry ecr-secret \
  --docker-server=YOUR_ACCOUNT_ID.dkr.ecr.ap-northeast-2.amazonaws.com \
  --docker-username=AWS \
  --docker-password=$(aws ecr get-login-password --region ap-northeast-2) \
  -n backend-api
```

### 2. CrashLoopBackOff

**증상**:
```
kubectl get pods -n backend-api
NAME                           READY   STATUS             RESTARTS   AGE
backend-api-xxx                0/1     CrashLoopBackOff   5          5m
```

**원인**:
- 애플리케이션 시작 실패
- 환경 변수 오류
- 리소스 부족

**해결**:
```bash
# 로그 확인
kubectl logs backend-api-xxx -n backend-api

# 이전 컨테이너 로그
kubectl logs backend-api-xxx -n backend-api --previous

# 리소스 확인
kubectl top pod backend-api-xxx -n backend-api

# 환경 변수 확인
kubectl exec backend-api-xxx -n backend-api -- env
```

### 3. Pending 상태

**증상**:
```
kubectl get pods -n backend-api
NAME                           READY   STATUS    RESTARTS   AGE
backend-api-xxx                0/1     Pending   0          5m
```

**원인**:
- 노드 리소스 부족
- PV 바인딩 실패
- NodeSelector 불일치

**해결**:
```bash
# Pod 이벤트 확인
kubectl describe pod backend-api-xxx -n backend-api

# 노드 리소스 확인
kubectl top nodes

# PVC 상태 확인
kubectl get pvc -n backend-api
```

## 모니터링 문제

### 1. ServiceMonitor가 타겟을 발견하지 못함

**증상**:
Prometheus Targets 페이지에서 "0/0 up"

**원인**:
- Label selector 불일치
- Service에 metrics port가 없음
- Namespace가 다름

**해결**:
```bash
# ServiceMonitor 확인
kubectl get servicemonitor -n monitoring

# Label 확인
kubectl get pods -n backend-api --show-labels
kubectl get svc -n backend-api -o yaml | grep -A5 ports

# 수동으로 metrics 테스트
kubectl port-forward -n backend-api svc/backend-api 8080:80
curl http://localhost:8080/actuator/prometheus
```

### 2. Grafana 대시보드 데이터 없음

**증상**:
대시보드에 "No data" 표시

**원인**:
- Data Source 설정 오류
- 메트릭 이름 변경
- Time range 오류

**해결**:
```bash
# Prometheus Data Source 확인
# Grafana UI → Configuration → Data Sources

# Prometheus에서 직접 쿼리
kubectl port-forward -n monitoring svc/kube-prometheus-stack-prometheus 9090:9090
# http://localhost:9090에서 쿼리 테스트

# Grafana 로그 확인
kubectl logs -n monitoring deployment/kube-prometheus-stack-grafana
```

## Secret 문제

### 1. ExternalSecret Sync 실패

**증상**:
```
kubectl get externalsecret -n backend-api
NAME                  STATUS              READY
backend-api-secrets   SecretSyncedError   False
```

**원인**:
- IRSA 권한 부족
- AWS Secrets Manager에 시크릿 없음
- SecretStore 설정 오류

**해결**:
```bash
# ExternalSecret 상세 정보
kubectl describe externalsecret backend-api-secrets -n backend-api

# ESO 로그 확인
kubectl logs -n external-secrets -l app.kubernetes.io/name=external-secrets

# AWS Secrets Manager 확인
aws secretsmanager describe-secret \
  --secret-id gympt/dev/backend-api \
  --region ap-northeast-2

# IAM 권한 확인
aws iam get-role --role-name gympt-dev-external-secrets
```

## 네트워크 문제

### 1. Service 연결 실패

**증상**:
```
curl: (7) Failed to connect to backend-api port 80: Connection refused
```

**원인**:
- Service selector 불일치
- Pod가 Ready 상태 아님
- NetworkPolicy 차단

**해결**:
```bash
# Service 확인
kubectl get svc backend-api -n backend-api -o yaml

# Endpoints 확인
kubectl get endpoints backend-api -n backend-api

# Pod selector 확인
kubectl get pods -n backend-api --selector=app=backend-api

# NetworkPolicy 확인
kubectl get networkpolicy -n backend-api
```

### 2. DNS 해석 실패

**증상**:
```
nslookup: can't resolve 'backend-api.backend-api.svc.cluster.local'
```

**원인**:
- CoreDNS 문제
- Service 없음
- Namespace 오류

**해결**:
```bash
# CoreDNS 상태 확인
kubectl get pods -n kube-system -l k8s-app=kube-dns

# Service 확인
kubectl get svc -n backend-api

# DNS 테스트
kubectl run -it --rm debug --image=busybox --restart=Never -- nslookup backend-api.backend-api.svc.cluster.local
```

## 리소스 문제

### 1. PodDisruptionBudget으로 노드 드레인 실패

**증상**:
```
error when evicting pod: Cannot evict pod as it would violate the pod's disruption budget
```

**원인**:
- PDB가 minAvailable 조건을 충족할 수 없음
- 노드에 모든 replicas가 집중됨

**해결**:
```bash
# PDB 확인
kubectl get pdb -n backend-api

# 임시로 PDB 삭제 (주의!)
kubectl delete pdb backend-api -n backend-api

# 노드 드레인
kubectl drain ip-10-0-11-123 --ignore-daemonsets --delete-emptydir-data

# PDB 재생성 (Argo CD가 자동으로 복구)
```

### 2. HPA가 스케일링하지 않음

**증상**:
```
kubectl get hpa -n backend-api
NAME          REFERENCE              TARGETS    MINPODS   MAXPODS   REPLICAS
backend-api   Deployment/backend-api <unknown>  2         10        2
```

**원인**:
- Metrics Server 없음
- CPU/Memory requests 미설정
- ServiceMonitor 문제

**해결**:
```bash
# Metrics Server 확인
kubectl get deployment metrics-server -n kube-system

# Pod metrics 확인
kubectl top pods -n backend-api

# HPA 상세 정보
kubectl describe hpa backend-api -n backend-api
```
